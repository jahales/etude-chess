# v0.3.0 — Complete the core loops

> The design for v0.3.0, written to be **decision-complete**: an implementing session should
> be able to build any item below without re-deriving design choices. Read
> [architecture.md](architecture.md) first (module map + cross-cutting rules), and follow
> [dev-workflow.md](dev-workflow.md) (branch per issue → TDD → `npm run verify` → PR → CI →
> merge). Owner decisions recorded 2026-07-19.

## Goal
Both modes execute *commit → grade → understand* and then drop the thread. v0.3 closes the
loop — **every flagged mistake gets a path back into practice** — and unifies the app
(one metric language, one friendly entry point).

## Engineering constraints (apply to every PR)
- Reducers stay **pure** and unit-tested; side effects live in the hooks (ADR 0015).
- **Engine/board sync by construction**: any engine result dispatched to a reducer carries
  the FEN it was computed for and is dropped if stale (see `SET_EVAL`/`SET_LINES` guards).
- **Scores are always White's perspective** in the UI (`whiteScoreLabel`, `whiteWinPercent`).
- Engines stay behind their ports (`Analyser`, `MaiaOpponent`); GPL artifacts arm's-length.
- Metrics describe *this game*; no speed metrics; no transfer claims (constitution §9/§12).
- Each user-visible flow gets an e2e (they skip cleanly when Maia nets are absent).

## 1. Persist coach data with each game *(prerequisite PR — do first)*
**What:** store the coach's knowledge with every finished Maia game so replay never
re-analyzes.
- Extend `StoredGame` (src/persist/db.ts): `coachLog: CoachEntry[]`, `evalByPly:
  (PositionEval | undefined)[]`, and `kind: 'game' | 'playout'` (default `'game'`).
  Dexie stores non-indexed fields automatically — **no schema/version bump needed**; treat
  the new fields as possibly-`undefined` on old records.
- Write them in `usePlaySession`'s persist effect. Positions are *not* stored — recompute by
  replaying `sanHistory` with chess.js.
- Add db helpers: `listGames(limit): StoredGame[]` (newest first), `getGame(gameId)`,
  `lastGame()`; all best-effort like the rest of db.ts.
**Accept:** a finished game round-trips with tiers + eval labels intact; old records load.

## 2. Landing redesign — mode cards + setup screens
**What:** replace the single long Home with a chooser (owner picked cards + setup screens).
- **Home = cards**: *Play a coached game* (vs Maia), *Study a master game*, *Your games*
  (library). Each card: one-line pitch + a **live stat** — e.g. "last game 86% · 4 played"
  (from db helpers), "3 games studied". Stats render blank-quietly when db is empty.
- Click → **focused setup screen**: Maia setup (colour/level, existing `MaiaSetup`) or the
  master-game picker (existing list). Back link to Home. `App.tsx` mode state grows:
  `home | maia-setup | maia | guess-pick | guess | library | replay`.
- **Settings move into context**: the analysis-settings gear renders only on guess screens
  (it configures guess-mode analysis); the eval toggle already lives in play mode. The
  topbar gear becomes the home of global settings (theme, item 9).
**Accept:** e2e navigates Home → each mode and back; no dead settings UI on non-guess screens.

## 3. Game library + replay (#39)
**What:** browse stored games; step through any of them with the coach data attached.
- **Library screen**: table of games (date, level, colour, result, accuracy, take-backs,
  `playout` badge). Newest first. Click → replay.
- **Replay screen**: board + move list (tier dots + per-move eval labels from stored data) +
  a cursor. Controls: click any move to jump; ‹ › buttons + ArrowLeft/ArrowRight keys.
  Positions recomputed from `sanHistory` once (memo). **No engine calls in replay v1** —
  stored data only. Show the stored coach entry (headline/detail) for the selected ply when
  one exists.
- **"Worth another look" becomes navigation**: entries in the post-game `Review` link into
  replay at that ply (game is already persisted by then).
**Accept:** e2e plays a short game, resigns, opens it from the library, arrows through, jumps
via a review entry.

## 4. Start from a position: play-it-out + opening picker (#41)
**What:** one plumbing change, two features.
- `playMachine` `NEW_GAME` gains optional `startFen`, `startSans: string[]` (seed history for
  Maia's planes), and keeps explicit `yourColor`. `initialPlayState.positions[0]` becomes the
  start FEN; `historyForMaia` already works from `positions`.
- **Play it out from here**: buttons in the post-game Review ("Play it out") and Replay
  screen ("Play from here") start a new Maia game from that position, same colour, same
  level, stored with `kind: 'playout'`.
- **Opening picker**: on the Maia setup screen, an optional opening select (reuse
  `src/content/openings.ts` — it already has named SAN lines; derive each opening's start
  position by replaying its moves). Default remains "Initial position".
**Accept:** reducer tests for NEW_GAME-from-position (incl. Black to move first); e2e plays
out from a review moment; picking "Sicilian Defense" starts the game after 1.e4 c5.

## 5. Guess-the-move loop closers
- **Drill your misses**: Summary gains "Drill your misses" when any attempt is Tier B/C —
  re-enters the play screen with a quiz of those positions (worst swing first, cap 5), same
  commit/reveal loop. New `sessionMachine` action `START_DRILL { items, game, sessionId }`
  (reuse `START_GAME`'s shape; drills persist attempts under the new sessionId like any
  session). After the drill, the summary shows the drill's results.
- **Reason reflected back**: the Reveal shows `state.reason` as `You said: "…"` alongside the
  coach's why (omit when empty), and `factBundleToText` gains a `My reasoning: …` line so the
  LLM clipboard handoff includes it.
- **Accuracy in the guess summary**: `summarize()` (src/domain/session.ts) adds `accuracy` =
  `meanAccuracy(attempt swings)`; show it in the Stat row (2 decimals, same as play mode).
**Accept:** unit tests for START_DRILL + summarize().accuracy; e2e drills a deliberate miss.

## 6. Reveal enrichment (both modes)
- **Click a line to walk it on the board** — in the guess reveal (`LinesPanel`) and play
  Show-me: clicking a line enters a *preview*: board shows the position after k moves of
  that PV, with ‹ › stepping and an obvious "back to game" affordance. **Display-only
  component state** (compute FENs from base fen + pv via chess.js); do not touch the
  reducers. Preview never mutates game state.
- **Richer play-mode "why"** (`src/domain/coach.ts`, pure): add — your hanging pieces after
  the move (have), opponent pieces *you could have won* (SEE on the best move when it's a
  capture), and "the engine's move was a check/capture" notes via chess.js flags. Keep it to
  ≤2 sentences; the deep answer stays behind Show-me.
**Accept:** unit tests for the added coach facts; e2e clicks a line and sees the board change
and restore.

## 7. Curated pack: 3 → ~15 annotated classics
**What:** hand-curated content, *not* the #40 database. Extend `PackGame` with optional
`annotations: Record<number, string>` (ply → note, shown at that reveal under the master's
move).
- Add ~12 public-domain classics (game scores are facts; **annotations must be our own
  prose** — short, ≤2 sentences, engine-checked). Candidates: Steinitz–von Bardeleben 1895,
  Lasker–Bauer 1889, Rubinstein–Rotlewi 1907, Réti–Bogoljubov 1924, Capablanca–Tartakower
  1924, Byrne–Fischer 1956, Morphy–Anderssen 1858 (pick one), Pillsbury–Lasker 1896,
  Anderssen–Zukertort 1869, Bird–Morphy 1858, Zukertort–Blackburne 1883, Botvinnik–
  Capablanca 1938.
- Verify each PGN parses and `heroColorFromResult` picks the winner; keep blurbs honest.
**Accept:** games.test.ts covers every pack entry (parses, has ≥20 quiz items); reveal shows
the annotation when present.

## 8. Theme toggle + dark retune (#42) + polish
- Tokens already exist (`:root` + `prefers-color-scheme` block in styles.css). Add
  `:root[data-theme='light']` / `:root[data-theme='dark']` overrides mirroring the palettes;
  a topbar toggle cycling **system → light → dark** (persist in `localStorage`, apply in
  `main.tsx` before render to avoid flash).
- **Retune the dark palette** (owner finds the current dark ground unpleasant): warmer/softer
  surfaces, clearer line contrast; keep both palettes AA-contrast for text. Adjust the
  board's square colours per theme via react-chessboard's `customDarkSquareStyle`/
  `customLightSquareStyle` so the board doesn't clash.
- Small consistency polish while there: spacing on the new screens, consistent card styles.
**Accept:** toggle persists across reload; both themes readable; e2e smoke passes in both.

## Non-goals (explicit)
- **No score preview while selecting a move in guess mode** — it lets you shop moves by eval
  and guts commit-before-reveal (ADR 0014). Scores appear at the reveal.
- **No #40 database work** (v0.4, needs its own spike). No hints. No mid-game reflection
  prompts. Nothing adaptive (ADR 0007).
- Replaying the same position is *recall*, not the §8 "fresh instances" ideal — fine for
  immediate corrective practice; theme-generalised drills wait for the content pipeline.
