## The headline decision (read this first)

The final review (ADR [0014](decisions/0014-v0.1.0-guess-the-move.md)) flipped the first two
releases:

- **v0.1.0 — coached guess-the-move on master games.** Every move of a classic game is a graded
  retrieval event (~25–35 per game vs ~3 per played game — ~10× the training density), it's the
  self-serve analog of *lessons* (the top activity in Southwick 2026 at 5.21 pts/hr), and it
  needs none of the risky components — just board + rules + PGN + Stockfish.
- **v0.2.0 — play vs client-side Maia**, with the *same* coaching loop pointed at your own
  games. Unchanged in design from ADR 0013; personalized leak analytics start here.

The deeper logic: the **coaching core is identical** in both (win%-swing grading,
commit-before-reveal, fact bundle, justification capture). v0.1.0 builds that core under the
cheapest possible harness; v0.2.0 attaches it to the Maia opponent. Playing is the material
*generator*; coached review is the *value* — in both releases.

## What v0.1.0 is

> **"Study a master's game — by playing it."** Take the winner's side of a curated classic. At
> each non-trivial move you **commit a move and a one-line reason**; the coach grades your choice
> by how much winning chance it keeps (an engine-equal alternative earns full credit — you're not
> being graded on mind-reading the master), then shows the master's move and *why* it works.

**In scope:**
1. **The game harness** — load a curated PGN, play one side, auto-play the opponent's replies;
   skip trivial decisions (early book moves, forced recaptures) via simple heuristics.
2. **Engine grading** — Stockfish (WASM Worker) evaluates each position with deterministic
   limits; your committed move is tiered by **win%-swing** (constitution §3: tier, don't rank;
   MultiPV so any tier-A move gets full credit).
3. **Coached, retrieval-first reveal** — after you commit: the master's move, the engine's view,
   and a **plain-language why** from the rules-based fact bundle (hanging piece, conceded square,
   big swing). Justification stored as telemetry (constitution §5).
4. **The game pack** — ~20–30 hand-picked **public-domain classics** (Morphy, Anderssen,
   Capablanca era — ideal for ~1200), with light metadata (opening, phase themes). Game scores
   are facts; annotations are ours, engine-derived.
5. **Honest scoring & session summary** — tier rates (A/B/C), the 2–3 biggest misses replayable,
   per-game and cumulative. **No speed metric** (constitution §9).
6. **Capture everything** — every commitment (position, your move, your reason, tier, engine
   best, master move, timestamp) in IndexedDB. Telemetry seed for later phases.

**Explicitly NOT in v0.1.0** (sequenced later): the Maia opponent and own-game review (v0.2.0);
mid-game "step back" intervention (v0.3); game import from Lichess/Chess.com (later alternate
source); curated drills; the hidden-mode mixed queue; the skill model; accounts/sync. **LLM**:
rules-based "why" + an optional **copy-the-fact-bundle-to-clipboard** handoff for pasting into
your own ChatGPT/Claude — no API integration yet (ADR [0012](decisions/0012-llm-grounded-explainer.md)).

**Why this is the right v0.1.0:** highest retrieval density per session of any mode we've
designed; the closest self-serve analog to the top-yield activity (lessons); zero build risk
beyond the already-derisked stack; a genuinely differentiated take on a format Chess.com users
keep asking for (existing tools grade match-the-master and capture no reasons); and every line of
it — grader, fact bundle, review UI, telemetry — is the core v0.2+ reuses unchanged.

## Technology selection

**Shape: a client-side browser app, no backend.** "Runs on my laptop now" (static build /
`localhost`) *and* "deploy to AWS later" (same bundle → S3/CloudFront), one codebase, no rewrite
(ADR [0009](decisions/0009-tech-stack.md)).

| Concern | Choice | License | Why |
|---|---|---|---|
| Language / build | **TypeScript + Vite** | MIT | Fast dev loop, static output |
| UI framework | **React** | MIT | Only actively-maintained *permissive* board lib is React-only |
| Board UI | **react-chessboard** v5 | **MIT** | Arrows, highlights, drag/click, promotion dialog built in |
| Rules / FEN / PGN | **chess.js** v1 | **BSD-2** | De-facto standard; PGN parsing for the game pack |
| Engine (referee) | **Stockfish 18 WASM** (`stockfish` npm), in a **Web Worker** | GPL-3 (arm's-length) | Same API local & browser; lite single-threaded → no COOP/COEP |
| Local storage | **IndexedDB** (via **Dexie**) | Apache-2.0 | Attempts, sessions, progress |
| Piece set | **Chessnut** (Apache-2.0) / Fantasy·Spatial·Celtic (MIT) / RhosGFX (CC0) | permissive | Avoids the CC-BY-NC / GPL Lichess-set trap |
| Game pack | Public-domain classics as PGN (Lichess masters DB / pgnmentor as sources) | facts / PD | Scores uncopyrightable; annotations engine-derived, ours |

**v0.2.0 adds:** **Maia-1 ONNX** via **onnxruntime-web** (weights GPL-3, arm's-length in its own
Worker; ort-web MIT) — the opponent. Client-side, still no backend (ADR
[0013](decisions/0013-v0.1.0-play-vs-maia.md)).

**Licensing stance:** our code stays **permissive/unencumbered**. v0.1.0 ships exactly **one GPL
artifact** (Stockfish WASM), kept arm's-length in a Worker, with its license text + a link to the
exact source build. Maia's GPL notice joins in v0.2.0. We avoid chessground/chessops (GPL) and
non-commercial piece sets. Full reasoning: ADR [0009](decisions/0009-tech-stack.md).

## Engine integration (unchanged — the thing worth getting right early)

One thin interface (ADR [0010](decisions/0010-engine-architecture.md)); Stockfish now, Maia in
v0.2.0, both behind it:

```ts
interface Engine {
  send(uci: string): void;                   // "position fen …", "go nodes N"
  onLine(cb: (line: string) => void): void;  // "info … score cp/mate", "bestmove …"
  quit(): void;
}
```
- **Grading module (written once, above the interface):** win% from `score cp` (Lichess const
  `0.00368208`); classify by **win% swing**; **deterministic limits** (`go nodes N` / fixed
  depth); **MultiPV** so any move within tier A of best earns full credit — the load-bearing
  detail that separates us from match-the-master graders. Study **Patzer** for thresholds.

## Milestones

- **M0 — skeleton:** Vite+React+TS, board renders, chess.js legal moves, click/drag.
- **M1 — engine core:** `Engine` interface + `StockfishWorkerEngine`; evaluate a FEN; win% +
  swing classifier with unit tests.
- **M2 — game harness:** load a PGN, play one side with auto-replies, trivial-move skipping,
  the commit-move-and-reason UI.
- **M3 — grading + reveal:** tier the commitment (MultiPV), reveal master move + engine view +
  rules-based fact-bundle "why"; persist attempts; "copy fact bundle" button.
- **M4 — game pack:** curate ~20–30 PD classics with light metadata; pack format + loader.
- **M5 — scoring & summary:** tier rates, biggest-miss replay, cumulative view; Stockfish GPL
  notice; polish.
- **(Parallel, anytime) Maia spike:** Maia-1 ONNX in a Worker producing a legal move from a FEN
  — retires v0.2.0's main risk without blocking v0.1.0.

Each milestone is independently demoable — stop and reassess after any.

## The 0.x sequence after this (subject to change)
- **v0.2.0** — **play vs Maia** + the same coached review on your own games; by-phase leak
  analytics begin (ADR [0013](decisions/0013-v0.1.0-play-vs-maia.md)).
- **v0.3.0** — mid-game "step back and analyze" intervention; tactics drills from *your own
  missed tactics* (fresh instances, untimed, retrieval-first).
- **v0.4.0** — opening *safety*: trap avoidance + structure/plan recognition from your leaks;
  FSRS/Leitner repertoire drilling.
- **v0.5.0** — endgame technique curriculum (tablebase-adjudicated).
- **v0.6.0+** — the hidden-mode mixed queue (the differentiator); dedicated play-it-out; then
  the skill model. Per [backlog.md](backlog.md) and constitution §11.
- **Any time** — game import (Lichess/Chess.com) as an alternate content source.

## Open decisions for this release
- **Which side to play** — winner's side only (simplest), or either side? Suggest winner-only.
- **Skip heuristics** — start quizzing at move ~8 (or first out-of-book), skip only-legal-move
  and obvious-recapture positions. Tune by feel.
- **Game pack curation** — which 20–30 classics; suggest starting from the standard instructive
  canon (Morphy's Opera Game onward) filtered for clear, theme-heavy play.
- **LLM** — clipboard fact-bundle handoff in v0.1.0; BYOK API / chess MCP later (ADR
  [0012](decisions/0012-llm-grounded-explainer.md)).
- **Desktop packaging (Electron)** — no; the browser app already runs locally.
